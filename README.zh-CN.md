# accelerated-learning

[English](README.md) | 简体中文

一个 Claude Code 插件，把 Claude 变成基于**持久化、网状知识地图**的苏格拉底式导师——而不是一门线性课程。

## 为什么

普通的问答式辅导（包括大多数"AI 导师"提示词）有三个具体的失败模式：

1. **困惑未解就自动推进。** 只要答对一题就被当作"可以进入下一节"，哪怕学习者心里还有没问出口的疑问。
2. **隐藏领域的全貌。** 主题按模型自选的顺序一个一个出现，学习者永远看不到"还有什么存在"——那些"未知的未知"。
3. **假设先验知识而不是检验它。** 自报的水平（"我对 X 算中级"）并不可靠；模型假设你懂的和你实际懂的很少吻合。

`accelerated-learning` 这个 skill 正是为了补上这三个缺口而存在的。

## 它做什么

- **诊断，而不是假设。** 开始教学之前，先在整个主题范围内做探测提问——每题作答*之前*先收集一个置信度估计——定位你真正掌握的内容，并呈现你的置信度与正确率之间的差距（校准线按最近 10 次探测计算，早期的失误不会永远拖累你）。探测题永不重复：重复的题目测的是对这段对话的记忆，不是理解。
- **立刻展示完整地图。** 说出一个新主题后，下一条回复就会给出初始化的知识图——8-15 个章节级节点，带类型化的边（前置 / 平行 / 应用），按声明的默认深度绘制、随时可以要求重新缩放——绝不会先用"你的背景是什么？"来拖延。已掌握、学习中、缺口、已知但未探索的节点全部可见；只有预埋的 `locked` 子主题保持隐藏，直到某次探测、一次"深入"或你自己的提问将其浮现。你问到的图上没有的概念会当场加入地图。
- **你来导航，而不是模型。** 每次节点转换时，模型都会给出一份明确的菜单（深入当前节点 / 转向兄弟节点 / 跳到相邻节点 / 查看地图 / 停止），而不是悄悄替你选好下一步。说"你来定"会得到一条带理由的推荐——同时其他选项会被重新列出，下一次转换时完整菜单照常出现。
- **入口要测，出口也要测。** 一个节点被标记为已掌握，必须同时满足：你明确确认没有剩余疑问，**并且**通过一道出口探测——一道全新的迁移题，先报置信度。"没有疑问"本身只是自我报告，这个 skill 在入口不信自我报告，在出口同样不信。出口探测答错则节点保持学习中，下次会换一道*不同的*出口题。
- **掌握会生疏——也会回退。** 已掌握的节点若约 30 天内没有答对过探测，恢复会话时会带上生疏标记，并附一次可选（绝不强制）的快速复检；复检失败节点回到 `gap`。即使没到生疏期限，后续探测答错或你表达出疑问，也会把已掌握节点退回学习中。地图永远不会对"你今天还能不能答对"撒谎。
- **引导但不磨人。** 在同一个点上错了 2-3 次、表现出明显挫败、或直接要答案时，它会直接给出该步骤的答案而不是没完没了地反问——带逃生阀的苏格拉底式教学。被告知答案绝不会让节点自动标记为已掌握；掌握之门依然生效。
- **跨会话持久化。** 进度按主题保存（一个主题一个文件，按语义匹配，所以"分布式共识"和 "distributed consensus" 永远不会分裂成两张图），每次节点转换后即写盘——不是只在会话结束时。恢复会话直接加载已保存的地图，不重跑诊断。每次会话以收尾结束：每个节点发生了什么变化、你的置信度-正确率校准线、已生疏的已掌握节点，以及一个下次的建议切入点。

## 使用方法

直接开始一段学习对话——说出你想学的主题，或提及之前学过的主题。当 Claude 识别到你想要带地图的引导式深入学习（而不是单次事实查询——那种它会直接回答）时，skill 就会被触发。

## 目录结构

```
skills/
  accelerated-learning/
    SKILL.md            # 工作流、决策流程图、规则
    graph-schema.md     # 知识图 JSON schema（v2）+ 示例
tests/
  pressure-scenarios.md # RED/GREEN 压力测试用例（已编写，尚未执行）
```

每个主题的进度持久化在**插件之外**，位于 `~/.claude/accelerated-learning/<topic-slug>/graph.json`（自动创建）——插件更新永远不会覆盖学习状态，同一张地图在任何项目里都能访问。
