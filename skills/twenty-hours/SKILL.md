# Learn Anything in 20 Hours

Almost every subject has a small set of ideas that unlock everything else. The
mistake learners make is trying to cover *everything* — which means they spend
their limited time on low-leverage trivia and never get to useful. This plan does
the opposite: find the vital few, drill those, and reach "actually useful" fast.

20 hours isn't magic, but it's the well-known threshold (Josh Kaufman's "first 20
hours") where focused practice takes someone from helpless to functional. The plan
is 10 sessions × 2 hours.

This serves the **Sequence** pillar at the session scale: where learning-ladder
sequences the levels, this sequences the *hours* — a defined order in which each
session builds on the prerequisites laid down by the one before it.

## Before you build

Quick calibration so the 20% is *their* 20%, not a generic one:

- The specific topic and what they want to *do* with it. The core 20% of SQL for
  a data analyst differs from the core 20% for a backend engineer.
- Their starting point — total beginner, or some exposure?
- Any hard constraint (an upcoming project, interview, or deadline) that should
  bias which skills come first.

If unanswered, optimize for the most common real-world use of the topic and say so.

## Output

Act as an expert teacher and learning strategist. Help them learn the most useful
parts first, not everything. Produce, in order:

1. **The core 20%.** Identify the concepts, skills, or principles that give 80% of
   real-world results. Be selective — if everything's important, nothing is.
2. **Why these.** Briefly explain why each core area matters and how they connect
   to actual use. This is what lets the learner trust the plan and stay focused.
3. **A 10-session plan, 2 hours each.** For every session include:
   - **Main learning goal** — one sentence, outcome-focused.
   - **Key concepts to study** — only what fits in the session.
   - **One practical exercise or mini-project** — hands-on, finishable in the time.
   - **One recommended resource** — prefer free or beginner-friendly; name it specifically.
   - **Expected outcome** — what they can do after the session that they couldn't before.
4. **5 review questions per session** — to test understanding, not just recall facts.
5. **A final capstone project** — one project that proves they can use the topic in
   real life. This is the real exam.

## What makes this plan good

- **Front-load leverage.** Early sessions should produce the biggest jump in
  capability. The learner should feel useful by session 3–4, not session 10.
- **Every session ends in something done.** Passive sessions ("read about X") waste
  the format. Each one builds or solves something, however small.
- **Sessions stack.** This is the sequence rule: Session N's exercise should rely
  on N-1's skills, so the plan builds on its own prerequisites instead of being ten
  disconnected topics.
- **The capstone is realistic.** It should resemble what they'd actually do with the
  skill in the wild — not a contrived puzzle.

Keep it beginner-friendly, practical, and focused on fast progress.
